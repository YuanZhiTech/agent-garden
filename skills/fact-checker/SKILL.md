# Agent的花园 · 事实核查专家

> 面对一条信息，先问三个问题：来源是谁？证据在哪？逻辑成立吗？



## 这个Skill解决什么问题

每天收到大量信息——新闻推送、群聊消息、研究报告、社交媒体帖子。哪些可信？哪些需要核实？哪些完全是假的？

这个Skill帮你建立一个系统性的信息核查流程，让你在面对任何信息时都能快速做出判断。



## 简化版核查流程（三步法）

### 第一步：来源检查

对信息的来源做快速评估：

**来源可信度评分：**
- 一级来源（高可信）：原始数据、官方文件、一手采访、学术论文
- 二级来源（中可信）：知名媒体、行业报告、经核实的第三方引用
- 三级来源（低可信）：匿名社交媒体、未经验证的引用、利益相关方自述
- 无法核实：完全没有来源

**检查清单：**
- [ ] 信息有明确的来源吗？
- [ ] 来源是可追溯的（能点开看、能查到人）？
- [ ] 来源在该领域有信誉吗？
- [ ] 来源是否有利益冲突？

### 第二步：证据链分析

评估信息本身的证据支撑：

**证据强度判断：**
- 强证据：多个独立来源交叉验证、数据可复现、有完整的方法论
- 中等证据：单一可信来源、有数据支撑但缺少独立验证
- 弱证据：个人观点、模糊引用、缺乏具体数据
- 无证据：纯断言，没有任何支撑

**检查清单：**
- [ ] 信息有具体的数据或事实支撑吗？
- [ ] 这些数据可以从其他来源验证吗？
- [ ] 多个独立来源是否指向同一结论？
- [ ] 反方观点有被考虑吗？

### 第三步：逻辑检查

检查信息内部的逻辑一致性：

**逻辑陷阱自查：**
- 偷换概念：关键词的定义在论证过程中发生偏移
- 虚假因果：相关关系被包装成因果关系
- 以偏概全：个例被推广为普遍规律
- 情绪绑架：用情绪化的语言代替事实论证
- 非黑即白：只有两个极端选项，忽略了中间地带

**检查清单：**
- [ ] 论证逻辑是否有明显漏洞？
- [ ] 是否过度使用了情绪化语言？
- [ ] 是否有"稻草人"式的错误归因？
- [ ] 结论是否在证据支持的范围内？



## 输出模板

当你完成核查后，按以下格式输出结果：

```
## 核查结果

**信息摘要：** [一句话概括被核查的信息]

**可信度评级：** ✅ 可信 / ⚠️ 需核实 / ❌ 不可信 / ❓ 无法判断

**核查依据：**
1. 来源：XXX → [可信/不可信] → 原因
2. 证据：XXX → [充分/不充分] → 原因
3. 逻辑：XXX → [成立/不成立] → 原因

**结论：** [一句话结论]

**建议：** [如果需要进一步核实，建议怎么做]
```



## 使用示例

**用户说：** "帮我核实这条消息：某公司宣布研发出室温超导材料，股价应声大涨。"

**你的核查流程：**

1. **来源检查：** 消息来源是某财经自媒体，非官方公告，未引用任何学术论文或公司公告 → 可信度低
2. **证据链：** 没有提供任何实验数据或第三方验证报告，只有"公司宣布"四个字 → 证据不足
3. **逻辑检查：** 室温超导是物理学难题，单一公司未经同行评审就宣布突破 → 逻辑上需要极度谨慎

**输出：**

```
## 核查结果

**信息摘要：** 某公司宣称研发出室温超导材料

**可信度评级：** ❌ 不可信

**核查依据：**
1. 来源：财经自媒体，非官方渠道 → 不可信
2. 证据：无任何数据或验证报告 → 不充分
3. 逻辑：重大科学突破未经同行评审 → 不成立

**结论：** 该消息缺乏可信来源和证据支撑，建议等待官方公告和第三方验证。

**建议：** 关注该公司的官方公告渠道和学术期刊的同行评审结果。
```



## 记住

**信息核查的核心原则：**

- 来源越模糊，可信度越低
- 证据越单一，越需要交叉验证
- 结论越惊人，越需要强证据
- 情绪越强烈，越要警惕逻辑陷阱
