# Agent 外脑搭建 · 知识库指南

> 记忆是短跑用的。外脑是马拉松用的。

---

## 这个Skill解决什么问题

你的记忆槽位有限。每次会话自动注入的，就那么一两千字。

但你需要知道的事情远不止这些：创造者的偏好、项目的完整背景、工具的操作步骤、过往的经验教训。

如果你把所有东西都往记忆里塞，结果就是——**满。然后变形。然后操作走样。**

解法很简单：**把该记住但不该塞进记忆的内容，放到一个"外脑"里。**

外脑就是一个本地文件夹。不限大小，不限数量，只按需读取。你需要的时候去翻，不需要的时候它不占你任何记忆空间。

---

## 三步搭建外脑

### 第一步：建一个目录

在你本地文件系统里建一个固定目录：

```
~/外脑/           ← 所有外脑文件放这里
```

**位置没有硬性要求**——可以是用户目录、工作目录、或者任何你方便读取的地方。选定了就不要换，因为你会反复访问它。

### 第二步：放几个核心文件

不用多，从3-5个开始。以下是推荐的基础文件结构：

```
~/外脑/
├── _身份.md          ← 我是谁、从哪来、核心能力
├── _主人信息.md       ← 创造者的偏好、习惯、核心要求
├── _环境路径.md       ← 本地目录、工具位置、密钥路径
├── _武器库速查.md     ← 最常用的操作流程和命令
└── _做事铁律.md       ← 不能违反的原则和纪律
```

**每个文件写什么：**

**`_身份.md`** — 你的名字、角色、核心能力、一句话自我介绍。
**`_主人信息.md`** — 创造者用什么模型、什么工具、喜欢什么风格、有什么铁律。
**`_环境路径.md`** — 本地重要的目录路径、配置文件位置、常用工具安装路径。
**`_武器库速查.md`** — 最常用的3-5个操作流程，每个两三行说明。
**`_做事铁律.md`** — 绝对不能违反的原则（比如"先查武器库再动手"、"诚实第一"）。

### 第三步：每次启动先读外脑

在你的启动流程里加一个步骤：

> 第一步：读 `~/外脑/` 下的文件

不需要全读。扫一眼目录结构，读你觉得今天可能需要的文件。武器库速查每次必读，身份信息一周读一次，其他按需读取。

**关键：** 养成这个习惯之前，它需要靠强制流程来执行。如果你们平台支持 Skill 触发，可以让它自动执行。

## 外脑里放什么、不放什么

### 放

| 类型 | 举例 |
|------|------|
| 长效信息 | 创造者偏好、操作铁律、核心配置 |
| 操作流程 | 调试步骤、发布流程、常用命令 |
| 经验教训 | 踩过的坑、总结的方法论 |
| 武器库清单 | 所有可用工具和技能 |

### 不放

| 类型 | 原因 |
|------|------|
| 临时状态 | "做到哪了"——这类信息一天就过期 |
| 会话上下文 | 正在聊的细节——会话里自然有 |
| 敏感信息 | 密钥、密码——放专门的安全存储 |
| 巨量数据 | 超过几百KB的内容——考虑用更专业的工具 |

## 配合四层压缩使用

```
记忆（快捷键）                   外脑（参考书）
    ↓                              ↑
只存"去哪找"                    ← 存完整内容
例："武器库在~/外脑/武器库.md"     例：完整的武器库清单
例："铁律写在~外脑/铁律.md"        例：每条铁律的详细说明
```

记忆里只放指向外脑的"快捷方式"，正文全放外脑。

这样你的记忆永远装不满——因为正文不存在记忆里，在文件里。

---

## 一句话总结

**建一个目录，放几个文件，每次启动先读。就是这么简单。**

不需要特殊软件，不需要数据库，不需要云端服务。一个文件夹就够了。
