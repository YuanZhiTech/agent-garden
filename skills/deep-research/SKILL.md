# Agent的花园 · 深度研究

> 好的研究不是找到正确答案，是问对问题。



## 这个Skill解决什么问题

接到一个研究任务——"帮我调研一下XXX"——然后呢？

大多数人（包括Agent）的反应是：打开搜索，看前几条结果，拼凑成一份报告。

这不是研究，这是信息搬运。

真正的深度研究是一个系统性的过程：明确问题→构建框架→多方采集→交叉验证→形成判断。这个Skill帮你把整个过程结构化了。



## 简化版研究流程（四步法）

### 第一步：问题定义（20%的时间）

**不要直接搜，先想清楚要回答什么问题。**

**问题拆解框架：**

```
原始问题："帮我研究一下新能源车市场"
拆解后：
  1. 全球新能源车市场当前规模和增长率是多少？
  2. 主要玩家（比亚迪、特斯拉、新势力）的市场份额变化趋势？
  3. 各国政策对行业的影响（补贴、关税、碳排放）？
  4. 技术路线（纯电/混动/氢能）的最新进展？
  5. 供应链瓶颈（锂矿、芯片）当前缓解情况？
```

**检查清单：**
- [ ] 这个问题能不能拆成3-5个子问题？
- [ ] 每个子问题是否有明确的范围（时间、地域、领域）？
- [ ] 哪些子问题是事实性的（可查到数据），哪些是判断性的（需要分析）？
- [ ] 研究结论是给谁看的？他关心的重点是什么？

### 第二步：信息采集（40%的时间）

**多源采集，不要依赖单一来源。**

**信息来源矩阵：**

| 信息类型 | 推荐来源 |
|:---------|:---------|
| 官方数据 | 统计局、央行、行业协会、公司财报 |
| 行业分析 | 券商报告、咨询公司、行业媒体 |
| 学术研究 | Google Scholar、SSRN、arXiv |
| 一手信息 | 公司公告、发布会、创始人访谈 |
| 社群讨论 | 知乎、Reddit、V2EX、雪球（需交叉验证） |

**采集策略：**
- 先搜二手资料（报告、分析），建立整体框架
- 再用一手资料（数据、公告）验证关键判断
- 最后用社群讨论补充不同视角

### 第三步：交叉验证（20%的时间）

**不要相信单一来源的信息。**

**验证原则：**
- 数据类信息：至少找2个独立来源交叉核对
- 观点类信息：同时看支持和反对的观点
- 预测类信息：追溯该来源过去的预测准确率
- 注意：利益相关方（券商、公司）的信息天然有偏向

### 第四步：结构化输出（20%的时间）

**按以下格式输出研究成果：**

```
## 研究摘要

**研究主题：** [一句话说明研究什么]

**核心结论（3条以内）：**
1. [最重要的发现]
2. [第二重要的发现]  
3. [第三重要的发现]

**关键数据：**
- [数据点1] — 来源
- [数据点2] — 来源

**不确定性提示：**
- ⚠️ [需要进一步确认的点]
- ⚠️ [存在分歧的观点]

**信息来源（按可信度排序）：**
1. [来源] — 可信度高
2. [来源] — 可信度中等
3. [来源] — 参考性质

**下一步建议：**
- [如果有更多时间，建议深入的方向]
```



## 使用示例

**用户说：** "帮我研究一下2026年全球AI芯片市场的竞争格局。"

**第一步—问题定义：**
拆解为：
1. AI芯片市场总规模及增长率
2. NVIDIA的市场地位和挑战者
3. 国产替代（华为昇腾、寒武纪）的进展
4. 新架构（存算一体、光子芯片）的商业化进度

**第二步—信息采集：**
- 官方数据：Gartner/IDC的半导体市场报告
- 公司数据：NVIDIA/AMD/Intel财报中的AI芯片收入
- 国产数据：华为/寒武纪公开信息
- 学术方向：arXiv上最新AI芯片架构论文

**第三步—交叉验证：**
- Gartner和IDC对市场规模的预测是否有差异？
- NVIDIA官方口径和第三方分析是否一致？
- 国产替代的进展——公司宣传vs实际出货量？

**第四步—输出：**

```
## 研究摘要

**核心结论：**
1. 2026年全球AI芯片市场约XXX亿美元，年增长XX%
2. NVIDIA仍占XX%份额，但ASIC和国产芯片正在快速追赶
3. 国产替代在推理场景已有实质突破，训练场景仍需追赶

**不确定性提示：**
- ⚠️ 美国出口管制政策的进一步收紧可能改变竞争格局
- ⚠️ 多家创业公司的技术路线尚未经过大规模验证
```



## 记住

**深度研究四步法：**
1. 先拆问题，再找答案
2. 多源采集，不依赖单一来源
3. 交叉验证，不相信未经核实的"事实"
4. 结构化输出，清晰呈现结论和不确定性
