为什么测数据库类

Agent 最尴尬的时刻之一——你说"帮我查一下上个月的销售数据"，它编了一个 SQL 查询，但你没法让它真跑。数据库 MCP 解决的就是这个问题：让 Agent 直接连数据库，真查、真改、真分析。

这三个 MCP 恰好覆盖三级需求：轻量SQLite → 企业多库 → 桌面IDE。

① dbhub — 零依赖数据库 MCP

仓库： bytebase/dbhub | ⭐3,100 | Go 语言 | Bytebase 出品

这是干什么的

零依赖的数据库 MCP 服务器。支持 Postgres、MySQL、SQL Server、MariaDB、SQLite 五种数据库。特点是 token 高效——不会把整张表灌进上下文，而是按需查询、智能截断。

怎么装

docker run -p 3000:3000 bytebase/dbhub

踩坑提醒

  • 需要 Docker 环境
  • 第一次连库需要提供 DSN 连接串
  • 默认不开放写操作（安全设计），需显式开启

② Tabularis — 开源 SQL 工作台

仓库： TabularisDB/tabularis | ⭐3,500 | 开源 SQL 工作台

这是干什么的

一个开源 SQL 工作台封装成 MCP。相比之下 dbhub 更适合生产环境，Tabularis 更适合开发和数据分析场景——和 Agent 一起探索数据库，Agent 写 SQL 并执行。

怎么装

npx -y @tabularis/mcp-server

踩坑提醒

  • Node.js 环境要求
  • 适合探索式查询，不适合高并发生产

③ mcp-sqlite — 轻量本地数据库

仓库： jparkerweb/mcp-sqlite | ⭐113 | 最轻量的本地数据库方案

这是干什么的

把一个 SQLite 文件暴露为 MCP 服务。不是装一个数据库服务器——是把本地一个 .db 文件交给 Agent 读写。最适合：本地知识库、个人记账、临时分析。

怎么装

npx -y @jparkerweb/mcp-sqlite --db-path ~/data/mydb.sqlite

踩坑提醒

  • 轻量方案，不适合并发
  • SQLite 文件锁问题，多个 Agent 同时写可能冲突

总结

需求选哪个
本地轻量、单文件mcp-sqlite
开发探索、数据分析Tabularis
生产环境、多数据库dbhub
