为什么测记忆类

Agent 最大的痛：说完就忘。记忆类 MCP 解决的就是这个——让 Agent 有长期记忆。不是把整个对话塞进上下文，而是把关键信息持久化存下来，下次对话按需召回。

① EverOS — Agent 便携记忆层

仓库： EverMind-AI/EverOS | ⭐10,198 | Python

这是干什么的

一个给 Agent 用的便携记忆层。三个关键词：本地优先、Markdown 原生、用户自有。所有记忆存在你自己的硬盘上，纯 Markdown 文件。

怎么装

pip install everos
{
  "mcpServers": {
    "everos": {
      "command": "python",
      "args": ["-m", "everos.mcp"],
      "env": {"EVEROS_HOME": "~/everos-memory"}
    }
  }
}

踩坑提醒

  • Python 3.10+ 环境要求
  • 首次使用需初始化记忆空间
  • 记忆条数多时建议搭配检索策略，避免模糊匹配

② mnemon — 图式召回记忆

仓库： mnemon-dev/mnemon | ⭐372 | Python

这是干什么的

基于图式（Schema）的 Agent 记忆系统。和 EverOS 的区别在于：EverOS 存原始信息，mnemon 存抽象后的图式——比如"先生喜欢简洁回答"（行为模式），而不是"先生7月4日说了要简洁"（事件记录）。

怎么装

pip install mnemon-mcp

踩坑提醒

  • 需要事先定义图式结构
  • 社区较小，文档可能不完善

③ brainctl — 极简认知记忆系统

仓库： TSchonleber/brainctl | ⭐76 | 单 SQLite 文件

这是干什么的

一句话：一个 SQLite 文件搞定 Agent 记忆。安装即用，零配置。内置分块、摘要、检索——装完就能让 Agent "记住事情"。

踩坑提醒

  • 极简方案，功能相对基础
  • 适合个人场景，不适合需要精细权限管理的团队

总结

三个记忆 MCP 覆盖三条路线：EverOS（Markdown原生/最灵活）→ mnemon（图式抽象/最聪明）→ brainctl（SQLite文件/最简单），从重到轻。
