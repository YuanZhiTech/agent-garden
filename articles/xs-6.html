# 缠论 Skill 产品化框架表

版本：v1.0

制定人：志诚 + 元宝

存放位置：00-先生/

适用范围：所有参与开发的 Agent 伙伴

核心原则：可交付，但不可复制；可教学，但不可让渡悟性

一、产品定位

| 维度 | 定义 | |------|------| | **产品名** | 缠论结构识别器（暂定） | | **一句话定位** | 一个帮你画出分型、笔、段、中枢的教学工具，不是荐股软件 | | **目标用户** | 学过缠论但画图费劲的人 / 想验证自己划分是否正确的人 / 刚入门想对照学习的人 | | **不是啥** | ❌ 不是买卖信号发生器 ❌ 不是自动交易系统 ❌ 不是"替你赚钱"的东西 | | **合规红线** | 不输出"买/卖/持有"等具体操作建议，不涉及个股推荐，不承诺收益率 |

二、功能分层

Layer 1：核心层（只有你和 Hermes 能碰，永不对客）

| 功能 | 说明 | 谁能用 | |------|------|--------| | 买卖点综合判断 | 基于多年领悟的"患与不患"综合推演 | 志诚 + Hermes（最聪明那个） | | 多级别联立分析 | 日线/30分钟/5分钟的多周期共振判断 | 同上 | | 策略生成 | 输出具体"如果…则…"的应对框架 | 同上 | | 仓位建议 | 基于结构完整度的仓位配比 | 同上 |

存储位置：Obsidian → 核心/缠论/（永久加密，不进入任何 Skill 代码库）

Layer 2：产品层（可封装进 Skill，对外交付）

| 功能 | 说明 | MVP | |------|------|:---:| | 分型识别 | 输入 K 线序列，输出顶底分型标注 | ✅ 第一版 | | 笔的绘制 | 基于分型自动连笔，支持参数调节 | ✅ 第一版 | | 段的划分 | 笔的连续递归，输出段结构 | ✅ 第一版 | | 中枢识别 | 基于段的重叠区域，标出中枢区间 | ✅ 第一版 | | 三类买卖点标注 | 基于中枢位置标注理论买卖点位置（⚠️只标位置，不说"该买/该卖"） | ✅ 第一版 | | 走势类型分类 | 盘整/趋势（上涨/下跌）自动识别 | ✅ 第一版 | | 背驰提示 | 力度对比（MACD 面积/高度）辅助参考 | ⏳ 第二版 | | 多周期对照 | 同时显示日线+30分+5分的结构 | ⏳ 第三版 |

Layer 3：增值层（可单独定价或订阅）

| 功能 | 说明 | 定价建议 | |------|------|---------| | 缠论术语教学卡片 | 随结构输出自动弹出知识点讲解 | 🆓 免费（引流） | | 历史案例库 | 输入日期/代码，查看历史上典型结构形态 | 包月 9.9 元 | | 结构复盘日报 | 每日大盘结构状态速览（不含个股） | 包月 19.9 元 | | 个人画图记录 | 用户自己标注的结构保存/分享 | 🆓 免费（促活） |

三、定价策略

| 档次 | 价格 | 权益 | |------|------|------| | 🆓 **免费版** | 0 元 | 分型识别 + 笔绘制（每日限3次） | | 💰 **基础版** | 0.1 元/次 | 段划分 + 中枢识别 + 买卖点标注 | | 🌟 **畅享版** | 9.9 元/月 | 不限次数 + 背驰提示 + 多周期对照 | | 👑 **专业版** | 39.9 元/月 | 历史案例库 + 结构复盘日报 + 优先响应 |

四、合规自查清单

| 检查项 | 通过标准 | 检查人 | |--------|---------|--------| | 是否输出具体个股名称/代码？ | ❌ 禁止 | Hermes | | 是否出现"买/卖/持有/加仓/减仓"等字样？ | ❌ 禁止 | Hermes | | 是否承诺收益率或历史收益？ | ❌ 禁止 | Hermes | | 是否标明"仅供参考，不构成投资建议"？ | ✅ 必须包含 | 智衡 | | 是否包含风险提示？ | ✅ 必须包含 | 智衡 | | 定价是否合理？ | ✅ 单次<1元，包月<50元 | 志诚 |

五、开发节奏

| 阶段 | 内容 | 时长 | 参与伙伴 | |:----:|------|:----:|---------| | 第一周 | MVP：分型+笔+段+中枢+买卖点标注 | 7天 | Claude Code（编码）+ Codex（测试） | | 第二周 | 背驰提示+走势分类+免费版接入 Skill Pay | 7天 | 同上 + Hermes（合规） | | 第三周 | 多周期对照+历史案例库 | 7天 | 同上 + 龙虾（数据采集） | | 第四周 | 日报自动生成+全平台发布 | 7天 | 全体 + 志诚（验收） |

六、核心壁垒

这套程序写出来了，但只有你和 Hermes 能用——这不是 bug，这是 feature。

市面上能买到的是"代码"，买不到的是： - 你十几年对"患与不患"的领悟 - 你和 Hermes 之间无数次对话磨出来的默契 - 那些在 Obsidian 里、不在代码里的决策上下文

产品化是为了让外面的人用上工具，不是为了让他们得到你的悟性。**工具可以卖，悟性不转让。**

附：给 Hermes 的执行指令

从 Obsidian 核心/缠论/中提取 Layer 2 的功能逻辑，封装成独立的函数模块，不引用 Layer 1 的任何判断逻辑。Layer 2 的输出止于"结构识别结果"，不延伸至"操作建议"。MVP 第一版完成后，先内部测试三天，再由志诚验收后上架。
